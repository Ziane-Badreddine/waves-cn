## Overview

`wave-cn.tsx` is the foundation of every waves-cn component. It exports:

- **`useWavesurfer`** — the primary React hook. Wraps wavesurfer.js with shared defaults, automatic CSS variable resolution, and stable memoization so your component never recreates the instance unintentionally.
- **`useCssVar`** — resolves `var(--*)` tokens at runtime so they work correctly with the Canvas API (which cannot read CSS variables directly).
- **`WAVESURFER_DEFAULTS`** — the shared default options applied to every instance. Override any value by passing it explicitly.
- **`WavesurferPlayer`** — a thin, memoized React component wrapper around the hook for declarative usage.

## Usage

### `useWavesurfer` hook

```tsx
import { useRef } from "react"
import { useWavesurfer } from "@/registry/hooks/audio"

export function MyPlayer() {
  const containerRef = useRef<HTMLDivElement | null>(null)

  const { wavesurfer, isReady, isPlaying, currentTime } = useWavesurfer({
    container: containerRef,
    url: "/audio/track.mp3",
  })

  return (
    <div>
      <div ref={containerRef} />
      <button onClick={() => wavesurfer?.playPause()} disabled={!isReady}>
        {isPlaying ? "Pause" : "Play"}
      </button>
    </div>
  )
}
```

### `WavesurferPlayer` component

```tsx
import WavesurferPlayer from "@/registry/hooks/audio"

<WavesurferPlayer
  url="/audio/track.mp3"
  waveColor="var(--primary)"
  height={80}
  onReady={(ws) => console.log("ready", ws)}
  onFinish={(ws) => console.log("finished")}
/>
```

### Overriding defaults

Any prop passed explicitly overrides `WAVESURFER_DEFAULTS`. Passing `undefined` gracefully falls back to the default.

```tsx
// Uses all WAVESURFER_DEFAULTS
useWavesurfer({ container, url })

// Override only what you need
useWavesurfer({ container, url, height: 120, barWidth: 5 })
```

### With plugins

Memoize the `plugins` array to prevent unnecessary instance recreation.

```tsx
import { useMemo } from "react"
import TimelinePlugin from "wavesurfer.js/dist/components/timeline.esm.js"

const plugins = useMemo(() => [TimelinePlugin.create()], [])

const { wavesurfer } = useWavesurfer({
  container,
  url,
  plugins,
})
```

## Defaults

All defaults are exported as `WAVESURFER_DEFAULTS` and can be imported to read or extend:

```ts
import { WAVESURFER_DEFAULTS } from "@/registry/hooks/audio"
```

| Option | Default | Description |
| ------ | ------- | ----------- |
| `waveColor` | `var(--muted-foreground)` | Waveform bar color. Resolved via `useCssVar` at runtime. |
| `progressColor` | `var(--primary)` | Progress overlay color. Resolved via `useCssVar` at runtime. |
| `height` | `64` | Waveform canvas height in px. |
| `barWidth` | `3` | Bar width in px. |
| `barGap` | `2` | Gap between bars in px. |
| `barRadius` | `2` | Border radius of each bar. |
| `minPxPerSec` | `1` | Minimum pixels per second (zoom level). |

## API Reference

### `useWavesurfer(options)`

```ts
import { useWavesurfer } from "@/registry/hooks/audio"
```

Accepts all `WaveSurferOptions` (except `container` which is passed as a `RefObject`) plus the `container` ref.

**Returns:**

| Property | Type | Description |
| -------- | ---- | ----------- |
| `wavesurfer` | `WaveSurfer \| null` | The wavesurfer.js instance. `null` until mounted. |
| `isReady` | `boolean` | `true` after the `ready` event fires. |
| `isPlaying` | `boolean` | `true` while audio is playing. |
| `hasFinished` | `boolean` | `true` after the `finish` event fires. |
| `currentTime` | `number` | Current playback position in seconds. |

---

### `WavesurferPlayer` (component)

```ts
import WavesurferPlayer from "@/registry/hooks/audio"
```

Accepts all `WaveSurferOptions` plus `on`-prefixed event handlers mapped from `WaveSurferEvents`:

| Prop pattern | Example | Description |
| ------------ | ------- | ----------- |
| `on{Event}` | `onReady` | Fires when wavesurfer emits the matching event. Receives `(wavesurfer, ...args)`. |
| `on{Event}` | `onFinish` | Same pattern for all wavesurfer events. |
| `on{Event}` | `onTimeupdate` | Called on every `timeupdate` tick. |

All `WaveSurferOptions` props are also valid — see [wavesurfer.js docs](https://wavesurfer.xyz/docs/modules/wavesurfer) for the full list.

---

### `useCssVar(value)`

```ts
import { useCssVar } from "@/registry/hooks/audio"
```

Resolves a CSS `var(--*)` token to its computed value at runtime. Required because the Canvas API cannot read CSS variables directly.

| Argument | Type | Description |
| -------- | ---- | ----------- |
| `value` | `string` | Any CSS value. If it matches `var(--*)`, it is resolved. Otherwise returned as-is. |

Returns the resolved color string (e.g. `hsl(222.2 47.4% 11.2%)` or `#3b82f6`).

Handles shadcn/ui's bare HSL channel format (`"222.2 47.4% 11.2%"`) by automatically wrapping in `hsl()`.

## Notes

### Why not use `@wavesurfer/react` directly?

waves-cn is inspired by [@wavesurfer/react](https://github.com/wavesurfer-js/wavesurfer-react) and follows the same hook-based architecture. The key differences are:

- **Shared defaults** via `WAVESURFER_DEFAULTS` — no need to repeat `height`, `barWidth`, etc. in every component.
- **Built-in CSS variable resolution** — `waveColor="var(--primary)"` just works.
- **`undefined`-safe merging** — passing `height={undefined}` gracefully falls back to the default instead of overriding it.
- **Tailored for shadcn/ui** — tokens, conventions, and component patterns match the shadcn ecosystem out of the box.

### Stable plugin references

Always memoize the `plugins` array. The hook compares the array reference — not deep equality — to decide whether to recreate the instance:

```tsx
// ✅ Stable — instance only created once
const plugins = useMemo(() => [ZoomPlugin.create()], [])

// ❌ New array every render — instance recreated on every render
useWavesurfer({ plugins: [ZoomPlugin.create()] })
```

### Imperative vs declarative options

Options that map to imperative wavesurfer methods should **not** be passed to `useWavesurfer` directly, as they will trigger instance recreation. Call them imperatively instead:

```tsx
// ❌ Causes re-creation on every speed change
useWavesurfer({ audioRate: speed })

// ✅ Imperative — no re-creation
wavesurfer?.setPlaybackRate(speed, preservePitch)
```
